# User Stories – Proyecto Web Corporativo  
**Empresa:** Multi Repuestos los Llanos C.A.  
**Proyecto:** Desarrollo de Página Web Corporativa  
**Duración:** 12 semanas  
**Tecnologías:** Next.js (App Router), React, TypeScript, Tailwind CSS  

---

## Épica 1: Planificación y Análisis del Proyecto (Semana 1)

### US-01 – Levantamiento de requerimientos del sitio web
**Como** pasante del departamento de desarrollo  
**Quiero** levantar los requerimientos funcionales y no funcionales del sitio  
**Para** definir correctamente el alcance del proyecto  

**Criterios de aceptación:**
- Se identifican las secciones principales del sitio web.
- Se documentan los requerimientos funcionales.
- El documento es validado por el tutor empresarial.

---

### US-02 – Análisis de sitios web de referencia
**Como** pasante  
**Quiero** analizar páginas web del sector  
**Para** identificar buenas prácticas de diseño y estructura  

**Criterios de aceptación:**
- Se revisan al menos tres sitios web similares.
- Se documentan elementos visuales y funcionales relevantes.
- El análisis sirve como referencia para el diseño del sitio.

---

## Épica 2: Arquitectura y Preparación del Entorno (Semana 2)

### US-03 – Definición de la arquitectura del sitio web
**Como** desarrollador  
**Quiero** definir la arquitectura del sitio web  
**Para** asegurar escalabilidad y mantenibilidad  

**Criterios de aceptación:**
- Se define la estructura de páginas y navegación.
- Se establece el uso de App Router.
- Se documenta el sitemap del sitio.

---

### US-04 – Configuración del entorno de desarrollo
**Como** desarrollador  
**Quiero** configurar el entorno de desarrollo con Next.js  
**Para** iniciar el desarrollo bajo estándares modernos  

**Criterios de aceptación:**
- El proyecto se crea con Next.js y TypeScript.
- Se configura Tailwind CSS.
- Se definen alias de rutas (`@/`).

---

## Épica 3: Diseño Visual y Maquetación (Semana 3)

### US-05 – Diseño visual del sitio web
**Como** usuario final  
**Quiero** un diseño visual profesional  
**Para** identificar a la empresa como una organización confiable  

**Criterios de aceptación:**
- Se define una paleta de colores corporativa.
- Se selecciona tipografía adecuada.
- El diseño es validado por el tutor.

---

### US-06 – Elaboración de prototipos visuales
**Como** desarrollador  
**Quiero** crear wireframes del sitio  
**Para** validar la estructura antes de la implementación  

**Criterios de aceptación:**
- Se elaboran bocetos de las páginas principales.
- Los prototipos reflejan la estructura del sitio.
- Se aplican ajustes según retroalimentación.

---

## Épica 4: Implementación del Frontend – Base del Sitio (Semana 4)

### US-07 – Implementación del layout principal
**Como** desarrollador  
**Quiero** implementar un layout base reutilizable  
**Para** mantener consistencia en todas las páginas  

**Criterios de aceptación:**
- Se implementa header, footer y contenedor principal.
- El layout se aplica a todas las rutas.
- El código sigue buenas prácticas.

---

### US-08 – Desarrollo de la página de inicio
**Como** visitante del sitio web  
**Quiero** visualizar la información principal de la empresa  
**Para** conocer rápidamente su actividad comercial  

**Criterios de aceptación:**
- La página muestra información clara y concisa.
- El diseño es responsivo.
- La navegación funciona correctamente.

---

## Épica 5: Implementación del Frontend – Contenido (Semana 5)

### US-09 – Desarrollo de la sección “Sobre Nosotros”
**Como** cliente potencial  
**Quiero** conocer la información institucional de la empresa  
**Para** generar confianza en sus servicios  

**Criterios de aceptación:**
- La sección presenta información institucional clara.
- El diseño mantiene coherencia visual.
- El contenido es accesible desde el menú principal.

---

### US-10 – Desarrollo de la sección “Servicios”
**Como** cliente o aliado comercial  
**Quiero** conocer los servicios que ofrece la empresa  
**Para** evaluar una posible relación comercial  

**Criterios de aceptación:**
- Se listan los servicios de forma clara.
- La página es responsiva.
- El contenido es fácil de comprender.

---

## Épica 6: Implementación del Frontend – Contacto y Responsividad (Semana 6)

### US-11 – Desarrollo de la sección de contacto
**Como** visitante del sitio web  
**Quiero** acceder a un formulario de contacto  
**Para** comunicarme con la empresa  

**Criterios de aceptación:**
- La sección de contacto está disponible.
- El formulario contiene campos obligatorios.
- El diseño es claro y usable.

---

### US-12 – Adaptación responsiva del sitio
**Como** usuario  
**Quiero** que el sitio sea completamente responsivo  
**Para** navegar desde cualquier dispositivo  

**Criterios de aceptación:**
- El sitio se adapta a móviles, tablets y desktop.
- No existen errores visuales.
- La experiencia de usuario es consistente.

---

## Épica 7: Backend y APIs (Semana 7)

### US-13 – Desarrollo del backend para el formulario de contacto
**Como** desarrollador  
**Quiero** crear un endpoint para el formulario  
**Para** procesar los mensajes enviados por los usuarios  

**Criterios de aceptación:**
- Se implementa una API Route en Next.js.
- Se validan los datos recibidos.
- La API responde correctamente al frontend.

---

## Épica 8: Integración del Sistema (Semana 8)

### US-14 – Integración frontend y backend
**Como** usuario  
**Quiero** que el formulario de contacto funcione correctamente  
**Para** enviar mensajes sin inconvenientes  

**Criterios de aceptación:**
- El formulario envía datos al backend.
- Se muestran mensajes de éxito o error.
- Se valida seguridad básica.

---

## Épica 9: Pruebas y Optimización (Semana 9)

### US-15 – Pruebas funcionales del sitio web
**Como** desarrollador  
**Quiero** realizar pruebas funcionales  
**Para** asegurar el correcto funcionamiento del sitio  

**Criterios de aceptación:**
- Todas las rutas funcionan correctamente.
- No existen errores críticos.
- Se corrigen fallos detectados.

---

### US-16 – Optimización de rendimiento
**Como** usuario  
**Quiero** que el sitio cargue rápidamente  
**Para** tener una mejor experiencia de navegación  

**Criterios de aceptación:**
- Se optimizan imágenes.
- Se reduce código innecesario.
- El tiempo de carga es aceptable.

---

## Épica 10: Documentación del Proyecto (Semana 10)

### US-17 – Documentación técnica del proyecto
**Como** desarrollador  
**Quiero** documentar la arquitectura y componentes  
**Para** facilitar el mantenimiento futuro  

**Criterios de aceptación:**
- Se documenta la estructura del proyecto.
- Se describen componentes y APIs.
- La documentación es clara y organizada.

---

## Épica 11: Despliegue del Sitio Web (Semana 11)

### US-18 – Despliegue del sitio en un entorno productivo
**Como** empresa  
**Quiero** publicar el sitio web corporativo  
**Para** hacerlo accesible al público  

**Criterios de aceptación:**
- El sitio está desplegado correctamente.
- Funciona en ambiente real.
- Se realizan ajustes finales post-despliegue.

---

## Épica 12: Presentación y Cierre del Proyecto (Semana 12)

### US-19 – Presentación final del proyecto
**Como** pasante  
**Quiero** presentar el proyecto finalizado  
**Para** demostrar el cumplimiento de los objetivos  

**Criterios de aceptación:**
- Se realiza una presentación formal.
- El sitio funciona correctamente.
- Se recibe aprobación de los tutores.

---

### US-20 – Entrega de informe final de pasantías
**Como** pasante  
**Quiero** entregar el informe final  
**Para** culminar formalmente las pasantías  

**Criterios de aceptación:**
- El informe incluye documentación técnica.
- Se adjunta el cronograma cumplido.
- El proyecto queda cerrado oficialmente.
